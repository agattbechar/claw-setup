## The Core Insight

**Most people download skills but don't understand them.**

They get:
- ✅ The artifact (the file)
- ❌ The reasoning (why it's structured that way)
- ❌ The mental models (what thinking it encodes)

**Result:** Slightly better "slop" — formatted garbage instead of plain garbage.

**For Bechar's Agency:** This explains why most AI agencies will fail. They'll download skills, apply them blindly, and produce generic work. The winners will understand the reasoning layer.

## The 3 Layers of Meta-Skills

| Layer | What It Is | Why Most Fail | How to Win |
|-------|------------|---------------|------------|
| **1. Trigger System** | When/why skill activates | Vague descriptions ("helps with writing") | Explicit boundaries: file types, trigger phrases, what it DOESN'T do |
| **2. Thinking Architecture** | How to THINK about problem class | Recipe-style: Step 1, Step 2, Step 3 | Restructure reasoning BEFORE output |
| **3. Verification Gate** | Check if output is differentiated | Generate and ship | Audit: "Does this look like default LLM output?" |

**For Viora:** Your AI avatar content needs all three layers:
1. **Trigger:** What prompts the avatar to create content?
2. **Thinking:** How does it approach viral content differently?
3. **Verification:** Does this look like every other AI video?

## The Contrarian Frame

**The Method:**
1. Write the "lazy/generic version" first
2. Name every predictable pattern
3. Engineer AWAY from each one

**Why It Works:**
- AI defaults to statistical average
- Average = forgettable
- Constraints force creativity

## Progressive Disclosure (Context Management)

**The Problem:** Context window is finite. Overload = quiet degradation (middle instructions get dropped).

**The Solution — 3 Tiers:**

| Tier | Purpose | Size | Example |
|------|---------|------|---------|
| **1. Always-On** | Routing logic, core workflow | <500 lines | "What phase are we in? What loads next?" |
| **2. On-Demand** | Deep knowledge, domain concepts | Variable | "Load reference/anti-patterns before delivery" |
| **3. Verification** | Final quality gate | Loaded last | "Does this look like baseline output?" |

**For Your Agency Skills:**
- Main skill file = router (lean)
- Reference files = domain knowledge (loaded when needed)
- Verification = loaded fresh before delivery

## The Expert Panel Problem

**Wrong Way:** "Evaluate your work as Expert X"
- AI roleplays a vague impression
- Pattern-matches to "what sounds smart"
- Catches nothing of substance

**Right Way:** Real cognitive profiles

**Extract from real work:**
- Recurring decision frameworks
- Prioritization logic (what do they look at FIRST?)
- Red flags (what makes them suspicious?)
- Sequence of questions before judgment
- What they ignore that others obsess over

**For Bechar:**
Build profiles of:
- Successful Mauritanian business owners (mom's network)
- Viral content creators in your niche
- People who actually convert customers

**Not:** "What would a marketer say?"
**But:** "Apply the prioritization framework of [specific person who grew from 0 to 100K followers]"

## The 7-Phase Meta-Skill Build

### Phase 1: Context Ingestion
- Dump everything you know about the problem
- Extract YOUR implicit methodology
- The way YOU do it manually = the gold

### Phase 2: Targeted Extraction (4 rounds max)
- **Round 1:** Scope (what should this accomplish? who uses it?)
- **Round 2:** Differentiation (what does AI get wrong? what's the lazy version?)
- **Round 3:** Structure (templates? multiple workflows? tools?)
- **Round 4:** Breaking points (what destroys a naive version?)

### Phase 3: Contrarian Analysis
- Write the generic version
- Challenge 2-3 assumptions
- Propose unexpected angles

### Phase 4: Architecture Decisions
- One task/minimal knowledge → One file, <300 lines
- One workflow/moderate depth → Modular (orchestrator + references)
- Multiple modes/deep knowledge → Full modular (workflow files, concepts, templates)

**Heuristic:** Main file >400 lines? Split it.

### Phase 5: Writing Content
- Build orchestrator FIRST (routes to everything)
- Every reference file needs EXPLICIT loading trigger
- Critical constraints at START and END of main file
- No hedge language ("always" not "try to")
- Every phase must yield visible output or concrete decision

### Phase 6: Real Review
Apply cognitive profiles:
- First-principles pass (does anything exist without earning its place?)
- Practicality check (would a real person use this daily?)
- Outcome check (does this shift behavior or just wrap process?)

### Phase 7: Ship
- Full file tree
- Architecture rationale
- Review findings
- Usage guide
